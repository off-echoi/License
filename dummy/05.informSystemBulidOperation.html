159. 소프트웨어 개발 방법론

  • 소프트웨어의 생산성과 품질 향상
  • 구조적 방법론, 정보공학방법론, 객체지향방법론, 컴포넌트기반 방법론, 애자일 방법론, 제품계열방법론

  1. 구조적 방법론

    • 정형화된 분석절차 처리(process)중심 방법론
    • 검증가능한 프로그램 코드생성 목적
    • 분할과 정복원리 적용
    • 타당성 검토 단계
      계획 단계
      요구사항 단계
      설계 단계
      구현 단계
      시험 단계
      운용/유지보수 단계

  2. 정보공학방법론

    • 정형화된 기법들을 상호연관있게 통합 및 적용 자료중심 방법론
    • 대규모 정보 시스템 구축
    • 정보 전략 계획 수립 단계
      업무 영역 분석 단계
      업무시스템 설계 단계
      업무시스템 구축 단계

  3. 객체지향 방법론

    • 개체를 객체로 만들어 부품조립하듯 소프트웨어 구현
    • 구조적기법의 문제점으로 인해 소프트웨어 위기의 해결책으로 채택
    • 구성요소 : 객체, 클래스, 메세지
    • 기본원칙 : 캡슐화, 정보은닉, 추상화, 상속성,다형성
    • 요구분석단계
      설계 단계
      구현 단계
      테스트 및 검증 단계
      인도단계

  4. 컴포넌트 기반 방법론

    • 컴포넌트 조합 > 애플리케이션 제작
    • 컴포넌트 재사용, 시간 노력 절감
    • 유지보수 최소화, 생산성 및 품질향상
    • 개발 준비 단계
      분석 단계
      설계 단계
      구현 단계
      테스트 단계
      전개 단계
      인도 단계

  5. 애자일 방법론

    • 짧은 일정주기 반복 개발
    • 소규모 프로젝트, 숙달개발자, 급변요구사항
    • XP,스크럼, 칸반, 크리스탈 등이 있음
    • 사용자 스토리
      2~4반복 계획
      개발
      승인테스트

  6. 제품계열 방법론

    • 특정제품에 적용하고자하는 공통된 기능 정읭하여 개발
    • 임베디드 개발 적합
    • 영역공학: 영역분석, 영역설계, 핵심자산구현
    • 응용공학: 제품요구분석, 제품설계, 제품구현
    • 특정분야의 영역 지식, 핵심자산의 개선
      영역공학 영역 분석 -> 영역 설계 -> 핵심 자산 구현
      ↓
      제품 요구사항 <->
      ↓       		↓
      제품아키텍쳐 <->
      ↓       		↓
      제품 조립생산
      ↓
      응용공학 제품 요구 분석 -> 제품 설계 -> 제품 구현

160. 비용 산정 기법

  1. 비용이 지나치게 높음 : 예산낭비, 효율성저하 / 낮음 : 품질문제, 개발자 부담
  2. 상향식, 하향식 산정기법

  3. 소프트웨어 비용 결정 요소

    • 개발하는 소프트웨어
    • 개발에 투입되는 자원
    • 소프트웨어 생산성

    1. 프로젝트요소

      제품 복잡도
      발생 가능한 문제의 난이도
      시스템 크기
      개발할 시스템 크기
      요구되는 신뢰도
      기간내에 조건하에 프로그램이 필요한 기능수행 정도

    2. 자원요소

      인적 자원
      개발관련자들의 자질
      하드웨어 자원
      장비, 프린터 등
      소프트웨어 장비
      언어분석기, 문서화 도구 등

    3. 생산성 도구

      개발자 능력
      개발자의 전문지식, 경험, 이해도, 창의력 등
      개발기간
      개발 소요 기간

비용 산정 기법

하향식 상향식
개요 과거 유사경험을 바탕으로 전문 개발자들이 회의를 통해 산정(비과학적) 프로젝트의 세부적인 작업단위별로 비용산정 -> 비용집계
기법

  1. 전문가 감정 기법

    • 조직내의 경험 많은 두명이상의 전문가에게 비용 산정 의뢰
    • 편리, 신속, 의뢰자의 신뢰
    • 새로운 프로젝트와 과거프로젝트와의 다른점 간과
    • 새로운 프로젝트와 유사한 경험이 없을수있다.
    • 개인적, 주관적

  2. 델파이기법

    • 많은 전문가 의견 종합
    • 한명의 조정자, 여러명의 전문가
    • 조정자 : 서식제공(시스템정의서, 비용내역)
      산정요원 : 익명 -> 비용산정
      조정자 : 요원 반응 요약, 배포
      산정요원 : 이전 참고 재산정(익명)
      의견이 거의 일치할때까지 반복

  1. LOC(원시코드라인수)기법

    • 비관치, 낙관치, 기대치를 측정하여 예측치를 구함
    • 측정용이
    •           a+4m+b
  예측치 = --------   a: 낙관치, b:비관치, m:기대치(중간치)
            6
      • 노력(인월) = 개발기간 * 투입인원 = LOC/1인당 평균생산코드라인수
      • 개발비용 = 노력(인월) * 단위비용(1인 평균 인건비)
      • 개발기간 = 노력(인원) * 투입인원
      • 생산성 = LOC / 노력(인월)
      • 예시 //// 답변
      • ㅁㅁㅁ

  2. 개발 단계별 인월수(Effort Per Task)기법

    • LIC를 보완, 각 기능을 구현시키는데 필요한 노력을 생명주기의 각 단계별로 산정

161. 수학적 산정기법

  • 상향식 비용산정기법, 개발비용 산정의 자동화를 목표
  • = 경험적 추정모형 = 실험적추적모형
  • 과거 유사한 프로젝트 기반으로 경험적 유도
  • COCOMO모형, Putnam모형, 기능점수 모형

  1. COCOMO 모형

    • 제안자 보헴, LOC에 의한 산정기법
    • LOC 예측 후 소프트웨어에따라 다르게 대입
    • 같은규모라도 성격게따라 다른 비용
    • 비용산정 결과 = ManMonth

    COCOMO의 소프트웨어 개발 유형

    1. 조직형(Organic Model)

      • 중소규모의 소프트웨어, 일괄 자료처리, 과학기술계산용 등 5만라인이하
      • 노력(MM) = 2.4 * KLOC
        개발기간 = 2.5 * (MM)

    2. 반분리형(Semi-Detached Model)

      • 30만 라인 이하, 컴파일러, 인터프리터같은 유틸리티 개발
      • 노력(MM) = 3.0 * KLOC
        개발기간 = 2.5 * (MM)

    3. 내장형(Embedded Model)

      • 30만 라인 이상, 신호제어기,실시간처리시스템 등 시스템 프로그램
      • 노력(MM) = 3.6 * KLOC
        개발기간 = 2.5 * (MM)

    COCOMO 모형의 종류

    1. 기본형 COCOMO (Basic)

      • 소프트웨어의 크기(LOC)와 개발 유형만을 이용하여 비용을 산정하는 모형

    2. 중간형 COCOMO (Intermediate)

      다음 4가지 특성의 15가지 요인에의해 비용산정

      제품의 특성
      : 요구되는 신뢰도, 데이터베이스 크기, 제품의 복잡도
      컴퓨터의 특성
      : 수행 시간의 제한, 기억저장소의 제한, 가상기계의 안정성, Turn Around Time
      개발 요원의 특성
      : 분석가의 능력, 개발분야의 경험, 가상 기계의 경험, 프로그래머의 능력, 프로그래밍 언어의 경험
      프로젝트의 특성
      : 소프트웨어 도구의 이용, 프로젝트 개발일정, 최신프로그래밍 기법의 이용

    3. 발전형 COCOMO(Detailed)

      • 중간형의 보완, 개발공정별로 보다 자세하고 정확하게 노력을 산출 하여 비용 산정
      • 소프트웨어의 환경과 구성요소가 미리 정의되어있어야함, 개발과정의 후반에 주로 적용

  2. Putnam 모형

    • 프로젝트 전반의 노력의 분포를 가정하는 모형
    • 생명주기 예측 모형
    • Rayleigh-Norden 곡선의 노력분포도를 기초로 한다.
    • 대형프로젝트
    • 개발기간이 늘어날수록 프로젝트 적용 인원의 노력 감소
    •                               원시 코드 라인 수
  개발노력(MM) = ----------------------------------------------
                환경상수(빈약=2,~/좋=8,~/최적=12,~) * 개발기간

  3. 기능점수(FP)모형

    • 알브레히트 제안
    • 소프트웨어의 기능을 증대시키는 요인별로 가중치
    • 요인별 가중치 합산 > 총기능점수산출 == 기능점수 > 비용 산정

  4. 자동화 추정 도구

    SLIM : Putnam, Rayleigh-Norden 기반
    ESTIMACS : FP기반
