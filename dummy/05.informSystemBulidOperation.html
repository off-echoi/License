159~160. 비용 산정 기법

하향식 상향식
개요 과거 유사경험을 바탕으로 전문 개발자들이 회의를 통해 산정(비과학적) 프로젝트의 세부적인 작업단위별로 비용산정 -> 비용집계
기법

  1. 전문가 감정 기법

    • 조직내의 경험 많은 두명이상의 전문가에게 비용 산정 의뢰
    • 편리, 신속, 의뢰자의 신뢰
    • 새로운 프로젝트와 과거프로젝트와의 다른점 간과
    • 새로운 프로젝트와 유사한 경험이 없을수있다.
    • 개인적, 주관적

  2. 델파이기법

    • 많은 전문가 의견 종합
    • 한명의 조정자, 여러명의 전문가
    • 조정자 : 서식제공(시스템정의서, 비용내역)
      산정요원 : 익명 -> 비용산정
      조정자 : 요원 반응 요약, 배포
      산정요원 : 이전 참고 재산정(익명)
      의견이 거의 일치할때까지 반복

  1. LOC(원시코드라인수)기법

    • 비관치, 낙관치, 기대치를 측정하여 예측치를 구함
    • 측정용이
    •           a+4m+b
  예측치 = --------   a: 낙관치, b:비관치, m:기대치(중간치)
            6
      • 노력(인월) = 개발기간 * 투입인원 = LOC/1인당 평균생산코드라인수
      • 개발비용 = 노력(인월) * 단위비용(1인 평균 인건비)
      • 개발기간 = 노력(인원) * 투입인원
      • 생산성 = LOC / 노력(인월)
      • 예시 //// 답변

  2. 개발 단계별 인월수(Effort Per Task)기법

    • LIC를 보완, 각 기능을 구현시키는데 필요한 노력을 생명주기의 각 단계별로 산정

161. 수학적 산정기법

  • 상향식 비용산정기법, 개발비용 산정의 자동화를 목표
  • = 경험적 추정모형 = 실험적추적모형
  • 과거 유사한 프로젝트 기반으로 경험적 유도
  • COCOMO모형, Putnam모형, 기능점수 모형

  1. COCOMO 모형

    • 제안자 보헴, LOC에 의한 산정기법
    • LOC 예측 후 소프트웨어에따라 다르게 대입
    • 같은규모라도 성격게따라 다른 비용
    • 비용산정 결과 = ManMonth

    COCOMO의 소프트웨어 개발 유형

    1. 조직형(Organic Model)

      • 중소규모의 소프트웨어, 일괄 자료처리, 과학기술계산용 등 5만라인이하
      • 노력(MM) = 2.4 * KLOC
        개발기간 = 2.5 * (MM)

    2. 반분리형(Semi-Detached Model)

      • 30만 라인 이하, 컴파일러, 인터프리터같은 유틸리티 개발
      • 노력(MM) = 3.0 * KLOC
        개발기간 = 2.5 * (MM)

    3. 내장형(Embedded Model)

      • 30만 라인 이상, 신호제어기,실시간처리시스템 등 시스템 프로그램
      • 노력(MM) = 3.6 * KLOC
        개발기간 = 2.5 * (MM)

    COCOMO 모형의 종류

    1. 기본형 COCOMO (Basic)

      • 소프트웨어의 크기(LOC)와 개발 유형만을 이용하여 비용을 산정하는 모형

    2. 중간형 COCOMO (Intermediate)

      다음 4가지 특성의 15가지 요인에의해 비용산정

      제품의 특성
      : 요구되는 신뢰도, 데이터베이스 크기, 제품의 복잡도
      컴퓨터의 특성
      : 수행 시간의 제한, 기억저장소의 제한, 가상기계의 안정성, Turn Around Time
      개발 요원의 특성
      : 분석가의 능력, 개발분야의 경험, 가상 기계의 경험, 프로그래머의 능력, 프로그래밍 언어의 경험
      프로젝트의 특성
      : 소프트웨어 도구의 이용, 프로젝트 개발일정, 최신프로그래밍 기법의 이용

    3. 발전형 COCOMO(Detailed)

      • 중간형의 보완, 개발공정별로 보다 자세하고 정확하게 노력을 산출 하여 비용 산정
      • 소프트웨어의 환경과 구성요소가 미리 정의되어있어야함, 개발과정의 후반에 주로 적용

  2. Putnam 모형

    • 프로젝트 전반의 노력의 분포를 가정하는 모형
    • 생명주기 예측 모형
    • Rayleigh-Norden 곡선의 노력분포도를 기초로 한다.
    • 대형프로젝트
    • 개발기간이 늘어날수록 프로젝트 적용 인원의 노력 감소
    •                               원시 코드 라인 수
  개발노력(MM) = ----------------------------------------------
                환경상수(빈약=2,~/좋=8,~/최적=12,~) * 개발기간

  3. 기능점수(FP)모형

    • 알브레히트 제안
    • 소프트웨어의 기능을 증대시키는 요인별로 가중치
    • 요인별 가중치 합산 > 총기능점수산출 == 기능점수 > 비용 산정

  4. 자동화 추정 도구

    SLIM : Putnam, Rayleigh-Norden 기반
    ESTIMACS : FP기반
2. IT프로젝트 정보시스템 구축 관리

166. 네트워트 관련 신기술

  1. IoT(Internet of Things, 사물인터넷)

    • 정보통신기술을 기반으로 실세계(Physical)와 가상세계(Virtual)의 사물을 연결 / 서비스기반기술
    • 유비쿼터스
    • 사물이나 사물 그 자체가 지능화, 사물과 사물사이의 지능통신(M2M)을 인터넷으로 확장 > 현실과 가상세계의 모든 정보와 상호 작용하는 IoT개념
    • 기술 : 스마트센싱, 유무선 통신 및 네트워크 인프라기술, 사물인터넷 인터페이스 기술, 사물인터넷 서비스기술
    • 개방형 아키텍쳐를 요하기때문에 정보공유 부작용최소화를 위해 정보보안기술적용이 중요

  2. M2M(Machine to Machine, 사물통신)

    • 무선통신을 이용한 기계와 기계사이의 통신
    • 전기/가스무선검침, 무선신용카드조회기, 버스운행시스템, 변압기원격감시 등 무선으로 상호 작용하는 기술
    • 부호 분할 다중 접속(CDMA), GMS, 무선데이터통신 등 무선 통신망 사용

  3. 모바일 컴퓨팅(Mobile Computing)

    • 휴대형 기기로 이동하면서 자유로이 네트워크에 접속하여 업무를 처리할 수 있는 환경
    • 휴대기기와 네트워크기술의 진화
    • 기업 - 비즈니스기술 극대화>경쟁력 확보 // 개인 - 삶의 질 향상

  4. 클라우드 컴퓨팅(Cloud Computing)

    • 중앙컴퓨터 자원 사용으로 어디서나 인터넷을 통한 컴퓨터작업 수행가능
    • 중앙컴퓨터 : 복수의 데이터 센터를 통합한 대형 데이터 센터, 컴퓨팅자원 보유
    • 사용자 : 기기만 있으면 업무수행 가능

    • 그리드 컴퓨팅과 비교

      그리드 컴퓨팅 : 여러대의 컴퓨터 분산처리 -> 우주 문제 등 대규모 연산에 사용
      클라우드 컴퓨팅 : 중앙의 대형 데이터센터를 순간순가 대여해줌

  5. 모바일 클라우드 컴퓨팅(MCC)

    • 모바일 기기로 클라우딩 컴퓨터 인프라 구성하는 ICT기술
    • 모바일 기기나 운영체제에 구애받지않고 이용가능
    • 이동성 + 클라우드의 경제성이 결합되어 시너지 기대

  6. 인터클라우드 컴퓨팅(inter-cloud Computing)

    • 각기 다른 클라우드 서비스나 자원 연동
    • 인터클라우드 서비스형태
      대등접속(Peering)
      클라우드 서비스 제공자 간 직접연계
      연합(Federation)
      자원공유 기반, 사용 요구량에 따른 동적 자원 할당지원 > 논리적으로 하나의 서비스제공
      중개(Intermediary)
      서비스 제공자를 통해 중개서비스 제공

  7. 메시 네트워크(Mash Network)

    • 차세대 이동통신, 홈네트워킹 등 특수목적을 위한 기술,대규모 디바이스의 네트워크 생성에 최적화
    • 블루투스 SIG에서 지원
    • 라우터를 기지국으로 사용 > 무선랜의 한계 극복 > 모든 구간을 동일한 무선망처럼 구성
    • 건물자동화, 센서 네트워크 등 IoT솔루션에 최적화

  8. 와이선(Wi-SUN)

    • 장거리 무선통신을 필요로하는 IoT서비스를 위한 저전력 장거리(LPWA) 통신기술
    • 검침분야에 유용(짧은시간 빈번한 데이터 전송), 낮은 지연속도, 메시기반확장성, 펌웨어업그레이드 용이
    • 고창군 적용 : IEEE 802.15표준, 900MHs,최대 전송속도 300kbps

  9. NDN(Named Data Networking)

    • 콘텐츠 자체의 정보와 라우터 기능만으로 데이터전송 수행(IP의 대체)
    • 콘텐츠중심네트워킹( CCN )과 같음
    • 헤시 테이블에 기반을 두는 P2P시스템처럼 정보와 라우터 기능만으로 목적지를 확정

  10. NGN(Next Generation Network, 차세대 통신망)

    • ITU-T에서 개발중, 유선망 기반 이동통신에서 제공한은 완전한 이동성 제공을 목표
    • 베경개념 : 하나의 망이 인터넷처럼 모든 정보와 서비스를 패킷으로 압축하여 전송
    • 인터넷 프로토콜기반 구축 all-IP

  11. SDN(Software Defined Network,소프트웨어 정의 네트워크)

    • 네트워크를 컴퓨터처럼 모델링 > 여러사용자가 각각의 소프트웨어로 네트워킹을 가상화하고 제어
    • 네트워크 비용 및 복잡성 해결 > 기존의 폐쇄형 기술을 개방형으로 변화시킬..

  12. NFC(Near Field Communication, 근거리 무선통신)

    • 고주파 이용 근거리 무선 통신
    • Ecma 340, ISO/IED18092표준, RFID기술의 일종
    • 13.56MHz 10cm 이내에 424Kbps 속도
    • 모바일 기기결제, 여행정보전송, 출입통제등 광범위하게 사용

  13. UWB(Ultra WideBand, 초광대역) = 디지털펄스

    • 짧은거리, 많은 데이터, 낮은 전력
    • 0.5m/W 의 저전력, 데이터 1km까지 전송, 땅/벽뚫기 지진, 재해시 전파탐지 인명구조 응용범위
    • 미국 군사목적으로 시작 2002상업화

  14. 피코넷(PICONET)

    • 여러개의 독립된 장치가 블로투스기능이나 UVW를 사용하여 통신망 형성
    • 상황에따라 조정프로토콜에의하여 마스터와 슬레이브의 역할로 네트워크형성
    • 좁은공간, 정지/이동장치 모두 포함

  15. WBAN(Wireless Body Area Network)

    • 웨어러블이나 삽입형 센서기기를 무선 연결
    • 수집정보를 휴대폰이나 간이형 기지국에 전달, uHealth의 서비스가능

  16. GIS(Geotraphic Information System, 지리정보시스템)

    • 위성으로 지리정보 제공
    • 단순 사물확인을 넘어 데이터 분석을 거쳐 최단기간들 제시

  17. USN(Ubiquitous SEnsor Network, 유비쿼터스 센서 네트워크)

    • 각종센서로 수집한 정보를 무선으로 수집가능하게 구성한 네트워크
    • 필요한 곳에 FFID 태그부착 후 주변정보탐지 > 네트워트에 연결하여 정보관리
    • 사람이 접근 불가한 곳에 설치하여 감시가능

  18. SON(Self Organizing Network, 자동 구성 네트워크)

    • 주변상황에 맞추어 스스로 망을 구성
    • 전송용량 확장의 경제성 문제 + 운영과 관리의 효율성
    • 갑작스러운 사용자의 증가나 감소시에 주변셀과의 협력을 통해 용량변화, 장애자체치유

  19. 애드 혹 네트워트(Ad-hoc Network)

    • 재난 상황처럼 유선망을 직접설치할 수 없을때, 모바일호스트만으로 구성한 네트워크
    • 망 구성 후 단기간사용, 유선망 구축이 어려운 상황에서 사용
    • 빠른 망 구성, 저렴한 비용

  20. 네트워크 슬라이싱(Network Slicing)

    • 글로벌 이동통신 표준화 단체가 선정한 5G 핵심기술
    • 하나의 물리적인 코어네트워크 인프라를 독립된 다수의 네트워크로 분리 > 각각의 네트워크는 고객맞춤형 서비스제공
    • VOD(주문형 네트워크) 형태로 AR, VR등 지원 기대
    • 소프트웨어 정의 네트워킹(SND)와 네트워크 기능 가상화(NFV)구현이 선행되어야 함

  21. 저전력 블루투스(BLE:Bluethooth Low Energy)

    • 연결되지 않은 상태에서는 절전 상태를 유지
    • 낮은전력으로 저용량데이터를 처리하는 극소형 사물 인터넷에 적합(비콘, 장난감, 시계)
    • 배터리하나로 몇년(전력효율 좋), 비용 효율

  22. 지능형 초연결망

    • 과기정통부 추진사업
    • 스마트 시티, 등 4차산업혁명에따라 증가하는 데이터 트래픽 수용을 위함
    • 국가전체망에 소프트웨어 정의 기술(SDE)적용, 데이터 트래픽증가를 야기하는 IoT, 빅데이터등을 수용
    • 중장기 네트워터 발전 전략

167. 네트워크 구축

네트워크 설치구조

  • 통신망 : 정보전달을 위해 연결한 통신설비의 집합
  • 성형, 링형, 버스형, 계층형, 망형

  1. 성형(star, 중앙 집중형)

    • 중앙 컴퓨터 중심으로 단말장치 연결
    • Point-to-Point 방식 연결
    • 각각의 통신은 중앙장치를 통해서 한다.
    • 단말 추가, 제거 용이
    • 중앙고장 = 전체 고장
    • 교환노드수 적다

  2. 링형(Ring, 루프형)

    • 서로 이웃 기기를 Point-to-Point 연결
    • 분산 및 집중 제어 가능
    • 추가/제거 어려움
    • 전송지연 발생가능
    • 중계기 수 많아짐
    • 단방향: 컴퓨터, 노드, 회선중 어느것이라도 고장나면 전체통신망에 영향
      앙방향: 우회가능

  3. 버스형(BUS)

    • 한개의 통신회선에 여러대의 단말장치
    • 기기고장이 통신망에 영향이 없어서 신뢰
    • 기밀보장 어려움, 통신회선 길이 영향

  4. 계층형(Tree, 분산형)

    • 중앙컴퓨터와 일정 지역 기기까지 하나의 회선
      이웃하는 단말은 중간단말로부터 다시 연결
    • 분산처리 시스템 구성

  5. 망형(Mesh)

    • 모든지점의 컴퓨터와 장치 연결, 노드의 연결성 높
    • 많은양의 통신, 공중데이터 통신만 사용, 통신회선의 총 경로가 길다.
    • 장애시 우회 가능
    • 노드가 n개일때 n(n-1)/2 개의 회선, 노드 당 n-1의 포트 필요

  6. 네트워크 분류

    • 지리적 범위에따라 LAN과 WAN으로 분류
    • 근거리 통신망(LAN : Local Area Network)
      회사, 학교..
      자원공유 목적
      속도빠름, 에러낮음
      주로 버스형이나, 링형
      광대역 통신망(WAN : Wide Area Network)
      국가와 국가...
      속도느림, 에러높음
      일정한 지역은 근거리로 묶고, 각 근거리를 연결
3. 소프트웨어 개발 보안 구축

179. Secure SDLC(소프트웨어 개발 라이프 사이클)

  • 보안상 안전한 소프트웨어를 개발하기위해 SDLC에 보안강화를 위한 프로세스를 포함한 것
  • 유지보수단계의 보안이슈 해결을 위함
  • CLASP, SDL
  • 요구사항분석, 설계, 구현, 테스트, 유집보수 등 SDLC 전체단계에 걸쳐 보안활동 제시

  1. 요구사항 분석에서의 보안 활동

    • 요구사항중 보안관련 식별
    • 조직의 보안정책참고, 적용할 수 있는 보안정책항목의 출처, 요구 수준, 세부내용문서화
    • 정보의 보안수준을 보안요소별로 등급을 구분하여 분류
      보안요소
      기밀성 (Confidentiality) 인가된 사용자만 접근, 전송중 노출되도 못 읽음
      무결성 (Integrity) 인가된 사용자만 수정가능
      가용성 (Availability) 인가사용자 언제든 이용
      인증 (Authentication) 사용자 합법적인지 확인 ex)비번, 지문
      부인방지 (NonRepudiation) 발뺌 못하게 송.수신 증거제공

  2. 설계에서의 보안 활동

    • 발생가능한 위협 식별 -> 보아대책, 소요예상, 영향범위 등 대책수립
    • 환경에대한 보안통제 기준을 수립
      네트워크 :
      외부 사이버공격대비 네트워크분리/방화벽
      서버 :
      보안에뛰어난 운영체체, 보안 업데이트, 외부접속통제
      물리적 보안 :
      출입통제, 개발공간 제한, 폐쇄회로설치
      개발 프로그램 :
      비인가 프로그램 제한, 무결성 검사

  3. 구현에서의 보안 활동

    • 표준 코딩 정의서, 소프트웨어 개발 보안 가이드 준수
    • 단위테스트를 통하여 보안취약점 개선
    • 코드점검, 소스진단코드 작업을통해 안정성 확보

  4. 테스트에서의 보안 활동

    • 설계단계의 설계서를 바탕으로 보안사항 점검
    • 동적분석도구 or 모의침투테스트를 통해 위협 해결여부 판단
    • 구현단계에서 추가된 위협 점검
    • 모든 결과는 문서화하여 보관, 개발자에게 피드백

  5. 유지보수에서의 보안 활동

    • 보안사고 식별, 보안패치 실행

188. 암호 알고리즘

  • 패스워드, 주민번호 등 중요정보를 보호하기 위해 평문을 암호화된 문장으로 만드는 절차/방법
  • 단방향 암호와 : Hash 사용
  • 양방향 암호와 : 개인키 및 공개키암호와
  •                                     ┌ Block 방식
                         ┌ 개인키 --ㄴStream 방식
              ┌ 양방향 --
              ㅣ         └ 공개키
암호와 방식 --
              └ 단방향 -- HASH

  1. 개인키 암호와(Private Key Encrytion)


    • 암호화 : 암호화알고리즘(개인키:평문) = 암호문
      복호화 : 복호와알고리즘(개인키:암호문) = 평문
    • 동일한 키로 데이터를 암호와, 복호화
    • = 대칭 암호화 기법 = 단일키 암호화 기법
    • 블록 암호화 :
      한번에 하나의 데이터 블록 암호화
      DES , SEED , AES , ARIA
      스트림 암호화 :
      평문과 동일한 길이의 스트림을 비트 단위로 암호화
      LFSR, RC4
    • 장점 : 속도가 빠름, 알고리즘 단순, 공개키보다 파일크기 작음
    • 단점 : 사용자 증가 = 관리할 키 증가

  2. 공개키 암호화(Public Key Encryption)


    • 암호화 : 암호화알고리즘(공개키:평문) = 암호문
      복호화 : 복호와알고리즘(비밀키:암호문) = 평문
    • 공개키 :
      데이터베이스 사용자에게 공개
      비밀키 :
      관리자가 비밀리에 관리
    • = 비대칭 암호화기법
      RSA기법, X509방식(인증서), 비X509방식(국가별)
    • 장점 : 키의 분배 용이, 관리할 키 적음
    • 단점 : 속도 느림, 알고리즘복잡, 파일크기 큼

  3. 해시(Hash) / 단방향

    • 임이의 입력데이터나 메시지를 고정된 길이의 값이나 키로 변환
    • 해시알고리즘 = 해시함수 / 변환된 값 = 해시값, 해시키
    • 암호화, 무결성 등 정보보호의 다양한 분야에서 활용
    • SHA시리즈, MD5, N-NASH, SNEFRU
4. 서비스 보안 구축

189. 서비스 공격 유형

  1. 서비스 거부(DoS:Denial of Service) 공격의 개념

    • 표적 서버의 정상기능 방해 : 대량의 데이터 한서버에 집중 전송(서버자원 고갈 목적)
    • PingOfDeth, SMURFING, SYN Flooding, TearDrop, Land, DDoS

      1. Ping of Death(죽음의 핑)

        • 패킷의 크기를 프로토콜 허용 범위 이상으로 전송 => 네트워크 마비

      2. SMURFING(스머핑)

        • 엄천난 양의 데이터를 전송 => 네트워크 불능
        • 네트워크 라우터의 브로드캐스트 주소를 수신지로 변경
        • 해결 : 네트워크 라우터에서 브로드태스트 주소를 사용할 수 없게 설정

      3. SYN Flooding

        • 서버가 대기상태에 놓여 정상적인 서비스를 수행 못하게 함
        • 사용불가 IP로 서버에 'STN'신호 전송 -> 서버는 'SYN+ACK'응답을 보내며 대기 공간 확보 -> IP는 'ACK' 전송 및 응답 불가의 상태로 서버는 계속 응답을 기다리며 무한대기
        • 해결 : 'SYN'수신 대기 시작을 줄임, 침입차단 시스템 활용

      4. TearDrop

        • 패킷 재조립의 오류로 과부하를 발생시킴
        • 패킷 분할 전송시 순서를 위해 Fragment Offset을 함께 전송 -> 중간에 Fragment Offset값을 변경하여 서버 오류
        • 해결 : Fragment Offset이 잘못된 경우 해당 패킷 폐기

      5. Land

        • 송신IP와 수신 IP주소를 같게 설정 => 자기자신에대한 무한 응답
        • 해결 : 송신IP와 수신IP의 적절성 검사

      6. DDoS(Distributed Denial of Service, 분산 서비스 거부) 공격

        • 분산된 공격 지점에서 한 곳의 서버로 분산 서비스 공격수행
        • 보안에 취약한 네트워크에 에이전트 설치 = 공격에 이용되는 컨퓨터가 됨
        • 분산 서비스 공격 툴 : Trin00, TFN, TFN2K, Stacheldraht

  2. 네트워크 침해 공격 관련 용어

    스미싱(smishing)
    • 문자 이용 -> 개인신용정보 탈취
    스피어 피싱(Spear Phishing)
    • 사회공학기법 => 특성대상 선정 , 지속적인 스팸메일
    APT
    (Advanced Persistent Threats, 지능형 지속 위협)
    • 조직적으로 특정 기업이나 네트워크 침투
    • 보안무력화, 정보수집 => 무력화
    • 내부자에게 악성 코드포함 이메일
    • 악성코드 Usb
    • P2P사이트 접속시 감염
    무작위 대입 공격
    (Brute Force Attack)
    • 암호화된 문서에 적용가능한 모든 값 대입
    큐싱
    • 큐알코드를 통해 악서앱 다운 유도
    SQL 삽입 공격
    • 전문 스캐너 프로그램으로 웹사이트 무작위 공격 => 취약사이트 발견 시 데이터베이스 조작
    크로스 사이트 스크립팅
    (XSS: Cross Site Scripting)
    • 스크립트 취약점을 악용한 해킹 기법
    • 특정 게시물 클릭시 사용자의 로그인정보, 개인정보 해킹

  3. 정보 보안 침해 공격 관련 용어

    좀비PC
    • 악성코드에 감염되어 다른 컴퓨터 조종, DDos에 이용
    C&C 서버
    • 좀비PC엣 명령을 내리는, 악성코드제어 서버
    봇넷
    • 악성프로그램이 깔려버린 컴퓨터들이 네트워크로 열결된 형태
    웜(Worm)
    • 연속적으로 자신을 복제 > 시스템 부학 높임 => 프로그램 다운
    • 분산서비스거부(DDos)공격, 버퍼오버플로우공격, 슬래머 등
    제로데이 공격
    • 보안취약점이 발견되자마자 공격, 공격의 신속성을 의미
    키로거공격
    • 키보드의 움직임 탐지, 개인정보 해킹
    랜섬웨어
    • 내부문서나 파일을 암호화, 해독조건으로 돈을 요구
    백도어
    • 시스템 설계자가 편의를 위해 만들어놓은 통로, 범죄에 악용
    트로이 목마
    • 정상프로그램인척 잠입, 동작할때 활성화되어 부작용

193. 보안 솔루션

  1. 방화벽(Firewall)

    • 전송정보를 선발하여 수용, 거부, 수정하는 기능을 가진 침입 차단시스템
    • 내부 -> 외부 : 통과
      외부 -> 내부 : 인증된 패킷만 통과

  2. 침입탐지 시스템(IDS:Instrucsion Detection System)

    • 컴퓨터의 비정상적인 사용, 오용, 남용탐지
    • 해커 침입패턴에 대한 추적과 유해정보 감시
    • 오용 탐지 : 미리 입력해둔 공격 패턴 감지
    • 이상 탐지 : 평균적인 시스템상태를 기준으로 비정상적인 행위나 사용 감지
    • 침입탐지 시스템의 위치
      패킷이 라우터로 들어오기 전
      네트워크에 시도되는 모든 공격 탐지
      라우터 뒤
      라우터에 의해 패킷 필터링을 통화간 모든 공격 탐지
      방화벽 뒤
      내부에서 외부로 향하는 공격 감지
      내부 네트워크
      내부에서 내부로의 해킹공격 탐지
      DMZ
      외부 인터넷 서비스를 제공하는 서버가 위치하는 네트워크
      공격으로 부터 중요데이터 보호, 서비스중단 방지

  3. 침입 방지 시스템(IPS : Instrucsion Prevention System)

    • 방화벽 + 침입탐지시스템
    • 비정상적인 트래픽을 능동적으로 차단, 격리
    • 침입탐지시스템 : 패킷 검사
    • 방화벽 : 비정상적인 패킷 차단

  4. 데이터 유출 방지(DLP : Data Leakage / Loss Prevention)

    • 내부정보의 외부유출 방지
    • 사내직원의 PC, 네트워크 등 사용자의 행위 탐지

  5. 웹 방화벽(Web Firewall)

    • 일반 방화벽이 감지하지 못하는 SQL삽입공격, Cross-siteScripting 등의 웹 기반 공격 방어 목적
    • 웹 관련 공격 감시, 웹 서버에 도달 전 차단

  6. VPN(Virtual Private Network, 가상 사설 통신망)

    • 가상 사설 네트워크 인터넷 등 통신사업자의 공증 네크워크와 암호화 기술을 이용 = 본인의 전용회선을 사용하는것처럼
    • 비용부담 줄임, 원격업무 수행

  7. NAC(Network Access Control)

    • 네트워크에 접속하는 내부 PC의 MAC주소를 IP관리 시스템에 등록한 후 일관된 보안관리 기능을 제공
    • 내부 pc의 소프트웨어 사용현황을 관리 => 불법적인 소프트웨어 설치 방지
    • 일괄적인 배포기능 => 백신이나 보안패치 일괄 설치,수행
    • 네트워크에 접속된 비인가 시스템 자동 차단

  8. ESM(Enterprise Securoty Managemant)

    • 다양한 장비에서 발생하는 로그 및 보안 이벤트 통합,관리
    • 방화벽, IDS,IPS, 웹방화벽 VPN에서 발생한 로그 및 보안 이켄트 통합, 관리
    • 종합적인 보완 관리 체계를 수립할 수 있다.
